🎧 The Web Audio API

How to get audio into JavaScript, and what "analysing" sound actually means.

Sound is Just Numbers

A microphone or audio file produces thousands of numbers per second — each one representing the air pressure at that moment. The Web Audio API is a set of JavaScript tools for working with those numbers: routing them, transforming them, and analysing them.

For a visualizer, we only need one part: the AnalyserNode. It sits in the audio path and measures what's passing through, so we can read the data and draw from it — without interrupting the sound itself.

The Audio Pipeline — Step by Step

Create an AudioContext

Every Web Audio program starts with an AudioContext. Think of it as the mixing desk — all audio flows through it. There should only be one per page. The browser now requires a user gesture (a click) before audio can start — this is a privacy/UX rule, not something we can override.

// Create the audio environment — do this once, on a button click
let ctx;

function initAudio() {
    if (ctx) return;  // already created — don't make a second one!
    ctx = new AudioContext();
}

💜 Why only one AudioContext?

Each AudioContext opens a connection to the system's audio hardware. Browsers limit this to a handful. Creating a new one every time you play something will exhaust the limit and cause errors. Create it once, reuse it everywhere.

Create an AnalyserNode

The AnalyserNode performs an FFT (Fast Fourier Transform) — a mathematical operation that converts the raw audio waveform into a list of frequency strengths. We set fftSize to control how detailed the analysis is. 256 gives 128 frequency bands — enough for a great-looking visualizer, and fast enough to not slow the browser down.

const analyser     = ctx.createAnalyser();
analyser.fftSize = 256;  // must be a power of 2: 32, 64, 128, 256, 512...

// frequencyBinCount = fftSize / 2 = 128 bands
const bufferLen = analyser.frequencyBinCount;  // 128

// Allocate the array that will receive frequency data each frame
// Uint8Array = 8-bit integers, values 0–255 (perfect for audio levels)
const freqData = new Uint8Array(bufferLen);

Connect an audio source

We use an <audio> HTML element as our source. We create a "media element source" node from it, then wire up the chain: source → analyser → destination. The destination is the speakers. If we didn't connect to the destination, the sound would be analysed silently.

// HTML: <audio id="player"></audio>
const audioEl = document.getElementById('player');

// Wrap the audio element in an AudioNode
const source = ctx.createMediaElementSource(audioEl);

// Chain: source → analyser → speakers
source.connect(analyser);
analyser.connect(ctx.destination);

⚠️ Connect an audio element only once

createMediaElementSource() permanently detaches the audio element from the browser's default audio path. If you call it a second time on the same element, you'll get a InvalidStateError. Use a flag (audioEl._connected = true) to remember whether you've already connected it.

Load a file with URL.createObjectURL()

When the user picks a file, we don't send it to a server — we create a temporary local URL that points to the file in memory. This is faster and works offline. The audio element can use this URL just like a normal web URL:

// HTML: <input type="file" id="picker" accept="audio/*">

document.getElementById('picker').addEventListener('change', e => {
    const file    = e.target.files[0];
    const blobURL = URL.createObjectURL(file);  // blob:// URL
    audioEl.src   = blobURL;
    audioEl.play();
});

Think of the AudioContext as a mixing desk in a recording studio. The source (your music file) plugs into a channel strip.   • What is a blob URL? — A temporary local URL that points to a file in memory, no server needed

📊 Reading Frequency Data

How sound is broken into bass, mids, and treble — and how to draw it.

What Does "Frequency" Mean?

Sound is vibration. Slow vibrations (low frequency) are bass — the thump of a kick drum, the rumble of a bass guitar. Fast vibrations (high frequency) are treble — the shimmer of a cymbal, the hiss of a hi-hat. The AnalyserNode measures how loud each frequency band is at this exact moment in the song.

With fftSize = 256, we get 128 frequency bands. Index 0 is the deepest bass; index 127 is the highest treble. Each value is between 0 (silent) and 255 (maximum volume). We call this 128-value snapshot once per animation frame.

Reading and Drawing Frequency Data

Call getByteFrequencyData() every frame

This method fills the freqData array with the current snapshot of frequency strengths. You must call it inside the animation loop — not once at setup — because the data changes continuously as the music plays:

function drawFrame() {
    // Snapshot current frequency data into our array
    analyser.getByteFrequencyData(freqData);

    // Now freqData[0]   ≈ bass loudness   (0–255)
    // Now freqData[64]  ≈ midrange loudness (0–255)
    // Now freqData[127] ≈ treble loudness  (0–255)

    drawBars();   // use the data to draw something

    requestAnimationFrame(drawFrame);   // schedule next frame (~60fps)
}

Convert a frequency value to a bar height

freqData[i] is 0–255. Our canvas is 300px tall. We divide by 255 to get a 0–1 ratio, then multiply by the canvas height to get pixels. This scales the data to fit whatever canvas size we're using — it works on any screen:

// freqData[i] is between 0 and 255
// Divide by 255 to get a ratio between 0.0 and 1.0
// Multiply by canvas height to get pixels
const ratio     = freqData[i] / 255;         // 0.0 to 1.0
const barHeight = ratio * canvas.height;       // 0px to 300px

// Bars grow upward from the bottom, so y = canvas.height - barHeight
const y = canvas.height - barHeight;

Draw all 128 bars in a loop

Divide the canvas width evenly between all bars, loop through every frequency band, and draw a rectangle for each one. We use hsl() colour with the hue based on the bar's position — this gives us a smooth rainbow across the frequency spectrum automatically:

function drawBars() {
    analyser.getByteFrequencyData(freqData);

    // Clear canvas
    ctx.fillStyle = '#000';
    ctx.fillRect(0, 0, canvas.width, canvas.height);

    // Each bar gets an equal slice of the total width
    const barW = canvas.width / bufferLen;

    for (let i = 0; i < bufferLen; i++) {
        const h    = (freqData[i] / 255) * canvas.height;
        const x    = i * barW;
        const y    = canvas.height - h;
        const hue  = (i / bufferLen) * 360;  // 0° = red … 360° = red again

        ctx.fillStyle = `hsl(${hue}, 100%, 50%)`;
        ctx.fillRect(x, y, barW - 2, h);
        // barW - 2 leaves a 2px gap between bars for readability
    }
}

  • Where does freqData[0] come from? — Index 0 is bass; the value is from getByteFrequencyData() called each frame
  • Why divide by 255? — To normalise the 0–255 range to 0.0–1.0 so it scales to any canvas height

🌈 Visualization Styles

The same 128 numbers, four completely different visuals.

One Data Source, Infinite Visuals

The frequency data array is just numbers. How we draw those numbers is entirely up to us — rectangles, lines radiating from a circle, a raw wave, or a particle system. All four modes you'll build here receive exactly the same data and produce wildly different results.

This is the creative heart of coding: the data doesn't change, but your interpretation of it can produce completely different art.

How Each Mode Works

Radial (Circle) Mode — frequency bars pointing outward

Instead of bars going straight up, we place each bar at a different angle around a centre point. We use Math.cos() and Math.sin() to convert the angle into x/y coordinates. The inner end of each bar is on the circle; the outer end extends by the frequency value:

const cx     = canvas.width  / 2;   // centre x
const cy     = canvas.height / 2;   // centre y
const innerR = 80;                   // radius of inner circle

for (let i = 0; i < bufferLen; i++) {
    // Spread bars evenly around the full circle (0 to 2π radians)
    const angle  = (i / bufferLen) * Math.PI * 2;
    const spike  = (freqData[i] / 255) * 150;  // max 150px spike

    // Inner point (on the circle)
    const x1 = cx + Math.cos(angle) * innerR;
    const y1 = cy + Math.sin(angle) * innerR;

    // Outer point (extended by spike length)
    const x2 = cx + Math.cos(angle) * (innerR + spike);
    const y2 = cy + Math.sin(angle) * (innerR + spike);

    ctx.strokeStyle = `hsl(${(i / bufferLen) * 360}, 100%, 55%)`;
    ctx.lineWidth = 3;
    ctx.beginPath();
    ctx.moveTo(x1, y1);
    ctx.lineTo(x2, y2);
    ctx.stroke();
}

Waveform Mode — the raw audio signal

The waveform shows the audio wave itself — not frequency content, but the actual pressure values over time. We use a different method, getByteTimeDomainData(), which fills an array with the current audio sample values (128 = silence, 0 = maximum negative, 255 = maximum positive). We draw them as a connected line path:

// Time domain data = the actual waveform shape
// Different from frequency data — size is analyser.fftSize, not frequencyBinCount
const waveData = new Uint8Array(analyser.fftSize);
analyser.getByteTimeDomainData(waveData);

ctx.beginPath();

for (let i = 0; i < waveData.length; i++) {
    // Normalise: 128 = centre line, 0 = bottom, 255 = top
    const v = waveData[i] / 128.0 - 1;  // -1 to +1
    const x = (i / waveData.length) * canvas.width;
    const y = (v + 1) / 2 * canvas.height;  // map -1..+1 to 0..height

    if (i === 0) ctx.moveTo(x, y);
    else         ctx.lineTo(x, y);
}

ctx.strokeStyle = '#43e97b';
ctx.lineWidth = 3;
ctx.stroke();

💡 Frequency vs. Waveform data

getByteFrequencyData() gives you the strength of each pitch. getByteTimeDomainData() gives you the raw audio pressure samples. Frequency data = "how much bass/treble right now?" Time domain data = "what does the wave actually look like?" Both are useful for different visual effects.

Particle Mode — objects that burst on beats

Particles are objects stored in an array. Every frame we create new particles where the frequency is loud enough, move existing ones by their velocity, fade them out by reducing their life value, and delete dead ones. The key pattern is iterating backwards when deleting from an array, so removed items don't cause index skipping:

let particles = [];

function drawParticles() {
    analyser.getByteFrequencyData(freqData);

    // Fade the background instead of fully clearing — creates trails
    ctx.fillStyle = 'rgba(0,0,0,0.2)';
    ctx.fillRect(0, 0, canvas.width, canvas.height);

    // Spawn new particles where audio is loud enough
    for (let i = 0; i < bufferLen; i += 4) {  // every 4th band
        if (freqData[i] > 140) {  // only spawn above threshold
            const speed = freqData[i] / 40;
            particles.push({
                x:    (i / bufferLen) * canvas.width,
                y:    canvas.height / 2,
                vx:   (Math.random() - 0.5) * speed,
                vy:   (Math.random() - 0.5) * speed,
                life: 1.0,
                hue:  (i / bufferLen) * 360
            });
        }
    }

    // Update & draw — iterate BACKWARDS when deleting from arrays!
    for (let i = particles.length - 1; i >= 0; i--) {
        const p = particles[i];
        p.x    += p.vx;
        p.y    += p.vy;
        p.life -= 0.018;

        if (p.life <= 0) { particles.splice(i, 1); continue; }

        ctx.fillStyle = `hsla(${p.hue}, 100%, 60%, ${p.life})`;
        ctx.beginPath();
        ctx.arc(p.x, p.y, 3 * p.life, 0, Math.PI * 2);
        ctx.fill();
    }
}

⚠️ Why loop backwards when deleting?

If you remove index 5 while looping forwards, the old index 6 slides down to become 5 — and your loop skips it entirely. Looping backwards means you always process the "later" items first, so removing an item never affects the indices of items you haven't processed yet.

✅ Before Moving On

  • How does Math.cos(angle) * radius place a point on a circle? — cos gives the x offset, sin gives the y offset at that angle
  • What's different about waveform vs frequency data? — Waveform = raw pressure samples over time; frequency = loudness by pitch right now
  • Why use rgba(0,0,0,0.2) to clear in particle mode? — Creates motion trails by only partially erasing the previous frame
  • Why iterate backwards when deleting from an array? — Prevents index skipping when items shift after a deletion

⚡PHASE 3 — JavaScript: The Logic

Setup: canvas, state variables, audio objects

The audio pipeline objects are declared as let at the top but initialised to null. We create them lazily — only when the user first interacts — because browsers require a user gesture before allowing audio.

inside <script> 
// Canvas
const canvas    = document.getElementById('myCanvas');
const ctx       = canvas.getContext('2d');

// Audio objects — created on first user interaction
let audioCtx   = null;
let analyser   = null;
let freqData   = null;
let bufferLen  = 0;

// Visualizer state
let mode         = 'bars';
let particles    = [];
let animId       = null;   // stores requestAnimationFrame ID
let isAnimating  = false;

const player     = document.getElementById('player');

Initialise audio — once, on first click

initAudio() creates the AudioContext and AnalyserNode if they don't exist yet. We call it from any function that needs the audio pipeline. The if (audioCtx) return guard means it's safe to call multiple times — it only creates the objects once:

inside <script> 
function initAudio() {
    if (audioCtx) return;  // already done — skip

    audioCtx         = new AudioContext();
    analyser         = audioCtx.createAnalyser();
    analyser.fftSize  = 256;
    bufferLen         = analyser.frequencyBinCount;  // 128
    freqData          = new Uint8Array(bufferLen);
}

function connectPlayer() {
    initAudio();
    if (player._connected) return;  // already wired up — don't do it twice!

    const source = audioCtx.createMediaElementSource(player);
    source.connect(analyser);
    analyser.connect(audioCtx.destination);
    player._connected = true;  // flag so we never call createMediaElementSource again
}

Mode setter and animation loop

When the user picks a mode, we cancel any existing animation loop (using the stored animId) and start a fresh one. This prevents the bug where multiple loops run simultaneously:

inside <script> 
function setMode(newMode) {
    mode      = newMode;
    particles = [];  // clear particles when switching modes

    // Update button styles
    document.querySelectorAll('#modeButtons button').forEach(b => {
        b.classList.toggle('active', b.textContent.includes(newMode));
    });
}

// The animation loop — runs ~60 times per second
function drawLoop() {
    if (!isAnimating) return;  // paused — stop scheduling frames

    switch (mode) {
        case 'bars':      drawBars();      break;
        case 'circle':    drawCircle();    break;
        case 'wave':      drawWave();      break;
        case 'particles': drawParticles(); break;
    }

    animId = requestAnimationFrame(drawLoop);  // schedule next frame
}

// Start/stop animation with the audio player
player.addEventListener('play', () => {
    connectPlayer();
    audioCtx.resume();
    if (!isAnimating) {
        isAnimating = true;
        drawLoop();
    }
});
player.addEventListener('pause', () => {
    isAnimating = false;
    cancelAnimationFrame(animId);
});

⚠️ The Multiple Animation Loops Bug

If you start a new loop without cancelling the old one, you get two loops running simultaneously — both trying to draw to the same canvas. This causes flickering and exponential performance degradation. Always cancel the old animId before starting a new loop, and use a boolean flag like isAnimating to stop loops cleanly.

    ctx.fillStyle = '#000';
    ctx.fillRect(0, 0, canvas.width, canvas.height);
    const bw = canvas.width / bufferLen;

        ctx.fillRect(i*bw, canvas.height-h, bw-2, h);
    }
}

function drawCircle() {
    analyser.getByteFrequencyData(freqData);
    ctx.fillStyle = 'rgba(0,0,0,0.15)';
    ctx.fillRect(0, 0, canvas.width, canvas.height);
    const [cx, cy, r] = [canvas.width/2, canvas.height/2, 80];
    for (let i = 0; i < bufferLen; i++) {
        const a   = (i / bufferLen) * Math.PI * 2;
        const len = (freqData[i] / 255) * 150;
        ctx.strokeStyle = `hsl(${(i/bufferLen)*360}, 100%, 55%)`;
        ctx.lineWidth = 3;
        ctx.beginPath();
        ctx.moveTo(cx+Math.cos(a)*r,        cy+Math.sin(a)*r);
        ctx.lineTo(cx+Math.cos(a)*(r+len), cy+Math.sin(a)*(r+len));
        ctx.stroke();
    }
}

function drawWave() {
    const wd = new Uint8Array(analyser.fftSize);
    analyser.getByteTimeDomainData(wd);
    ctx.fillStyle = '#000';
    ctx.fillRect(0, 0, canvas.width, canvas.height);
    ctx.strokeStyle = '#43e97b';
    ctx.lineWidth = 3;
    ctx.beginPath();
    wd.forEach((v, i) => {
        const x = (i / wd.length) * canvas.width;
        const y = (v / 128.0) * canvas.height / 2;
        i === 0 ? ctx.moveTo(x, y) : ctx.lineTo(x, y);
    });
    ctx.stroke();
}

function drawParticles() {
    analyser.getByteFrequencyData(freqData);
    ctx.fillStyle = 'rgba(0,0,0,0.2)';
    ctx.fillRect(0, 0, canvas.width, canvas.height);
    for (let i = 0; i < bufferLen; i+=4) {
        if (freqData[i] > 140) {
            const spd = freqData[i] / 40;
            particles.push({ x: (i/bufferLen)*canvas.width, y: canvas.height/2,
                vx: (Math.random()-.5)*spd, vy: (Math.random()-.5)*spd,
                life: 1, hue: (i/bufferLen)*360 });

    }
    for (let i = particles.length-1; i >= 0; i--) {

        p.x += p.vx; p.y += p.vy; p.life -= 0.018;

        ctx.beginPath();

        ctx.fill();

Draw the idle state — something nice before music loads

When no music is playing, the canvas should show something rather than a black void. A simple animated idle pattern makes the tool feel polished:

inside <script> 
function drawIdle() {
    ctx.fillStyle = '#0a0a0f';
    ctx.fillRect(0, 0, canvas.width, canvas.height);

    const cx = canvas.width / 2, cy = canvas.height / 2;
    const t  = Date.now() / 1000;  // seconds since page load

    // Gently pulsing rings
    for (let r = 20; r < 160; r += 28) {
        const pulse   = Math.sin(t * 1.5 + r * 0.04) * 0.5 + 0.5;
        const hue    = (r / 160) * 120 + 120;  // green to cyan
        ctx.strokeStyle = `hsla(${hue}, 100%, 60%, ${pulse * 0.5})`;
        ctx.lineWidth = 2;
        ctx.beginPath();
        ctx.arc(cx, cy, r + pulse * 10, 0, Math.PI * 2);
        ctx.stroke();
    }

    ctx.fillStyle = 'rgba(255,255,255,0.35)';
    ctx.font = 'bold 16px Arial';
    ctx.textAlign = 'center';
    ctx.fillText('Upload a music file to start the visualizer', cx, cy + 180);
    ctx.textAlign = 'left';

    requestAnimationFrame(drawIdle);  // keeps animating until music starts
}

drawIdle();  // start idle animation immediately on page load

🎉 Common Things That Go Wrong

  • Silence but no error: AudioContext is suspended — call audioCtx.resume() on a user click
  • "InvalidStateError" on connect: You called createMediaElementSource() twice — use the _connected flag
  • Canvas flickering wildly: Multiple animation loops running — cancel old animId before starting a new loop
  • Nothing drawn when switching modes: Old loop is stopped but new one never started — ensure isAnimating is true and call drawLoop()
  • Performance slows over time (particles): Particles array is growing but splice is broken — check you're looping backwards

Challenge 1 — Beat detector

Add a beat detection effect: average the bass frequencies (indices 0–10). If the average jumps more than 20% above the previous frame's average, flash the canvas background briefly. This creates a strobe-like effect locked to the kick drum.

Challenge 2 — Colour scheme picker

Add buttons that change the hue offset for all visualizations — a "warm" scheme (+0°), "cool" scheme (+180°), and "monochrome" scheme. Store the offset in a variable and add it to all hue calculations.

Challenge 3 — Symmetric bars

Make the bars draw from the middle of the canvas outward — up and down — by also drawing a mirrored bar below the centre line. This gives a classic symmetric equalizer look.

Challenge 4 — Sensitivity control

Add a slider that controls a multiplier (0.5 to 3.0) applied to all frequency values. Math.min(255, freqData[i] * sensitivity). Low sensitivity shows only loud sounds; high sensitivity makes even quiet music look dramatic.

📊 Reading Audio Data

// Frequency data — loudness of each pitch (0–255)
analyser.getByteFrequencyData(freqData);
// freqData[0]   = bass      (deepest)
// freqData[64]  = midrange
// freqData[127] = treble    (highest)

// Time domain data — raw audio wave shape (0–255)
const waveData = new Uint8Array(analyser.fftSize);
analyser.getByteTimeDomainData(waveData);
// waveData[i] = 128 → silence (centre)
// waveData[i] = 255 → maximum positive pressure
// waveData[i] = 0   → maximum negative pressure

🎨 Drawing from Audio Data

// Bar height: normalise 0–255 to canvas height
const barH = (freqData[i] / 255) * canvas.height;
const y    = canvas.height - barH;     // bars grow upward

// Rainbow colour by bar index
const hue = (i / bufferLen) * 360;
ctx.fillStyle = `hsl(${hue}, 100%, 50%)`;

// Radial: place bars on a circle
const angle = (i / bufferLen) * Math.PI * 2;
const x     = cx + Math.cos(angle) * radius;
const y     = cy + Math.sin(angle) * radius;

// Fade background instead of clearing (creates trails)
ctx.fillStyle = 'rgba(0,0,0,0.2)';
ctx.fillRect(0, 0, canvas.width, canvas.height);

⚡ Animation Loop Pattern

let animId      = null;
let isAnimating = false;

function startLoop() {
    if (isAnimating) return;  // prevent double-start
    isAnimating = true;
    loop();
}

function stopLoop() {
    isAnimating = false;
    cancelAnimationFrame(animId);
}

function loop() {
    if (!isAnimating) return;  // stop condition
    draw();
    animId = requestAnimationFrame(loop);
}

⚠️ The Four Most Common Bugs

  • AudioContext is suspended: Call .resume() inside a click/play handler — the browser blocks audio until user interaction
  • createMediaElementSource called twice: Throws an error and breaks the audio. Use a _connected flag to guard it
  • Multiple animation loops running: Store the rAF ID and call cancelAnimationFrame(animId) before starting a new loop
  • Particles array grows forever: Loop backwards (for (let i = arr.length-1; i >= 0; i--)) when splicing
