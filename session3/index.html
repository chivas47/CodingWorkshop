🖋️ The Problem We're Solving

One of the biggest challenges in tattooing: a client sees a design on paper and it looks very different when it's on their skin. Placement, scale, and curvature all change how a tattoo looks in real life.

Tattoo artists currently sketch on skin with a marker or use transfer paper just to show placement. We're going to build a browser tool that does this digitally — the client uploads a photo, picks a design, drags it around, and immediately sees how it looks.

🌍 Designing for Everyone

Skin comes in a huge range of tones — from the very lightest to the very darkest. Our tool will include 6 different skin tone placeholders based on the Fitzpatrick scale, which dermatologists use to classify skin types. This matters: tools that only show one skin tone are less useful for most of the world's population. Good software is inclusive by design.

🖼️ Images on Canvas

How to load a photo into JavaScript and draw it onto a canvas.

The Canvas Can Draw Images Too

So far you've used canvas to draw shapes and lines. But ctx.drawImage() lets you draw an entire photo or image file onto the canvas — like pasting a picture onto your drawing surface.

Once an image is on the canvas, you can draw on top of it — which is exactly how our tattoo visualiser will work: photo on the bottom layer, tattoo design on top.

How drawImage() Works — Step by Step

Create an Image object in JavaScript

Before you can draw an image, you need to load it into memory. JavaScript has a built-in Image constructor — it works exactly like creating an <img> element in HTML, but you do it entirely in JavaScript without touching the HTML file.

// This is equivalent to writing <img> in HTML, but in JS
let img = new Image();

// Tell it where to find the image file
img.src = 'photo.jpg';

💜 Why not just put <img> in the HTML?

We could add an <img> tag in HTML. But then we can't manipulate the photo with canvas tools — we can't draw on top of it, change its opacity, composite it with other images, or save the result. Creating an Image in JS and drawing it to the canvas gives us complete control.

Wait for the image to finish loading with onload

Here's a mistake that bites almost every beginner: images don't appear instantly. Even local images take a brief moment for the browser to decode them. If you call drawImage() before the image has finished loading, the canvas stays blank — no error, just nothing.

The solution is the onload event. You give the image a function to call when it's ready, and only inside that function do you draw:

let img = new Image();

// Step 1: set up what to do WHEN the image is ready
img.onload = function() {
    // This code only runs AFTER the image has fully loaded
    ctx.drawImage(img, 0, 0);
    console.log('Image drawn!');
};

// Step 2: THEN set .src to start loading
img.src = 'photo.jpg';

⚠️ Order Matters — Set onload BEFORE src

Think of it like this: setting img.src fires the starting gun. If you haven't set up your event listener yet, you'll miss the race. Always assign .onload first, then set .src to trigger the load.

Draw the image with drawImage()

ctx.drawImage() accepts the image and coordinates. The simplest form places the image at coordinates (x, y) at its natural size. The extended form lets you also specify the width and height — very useful for scaling a photo to fill the whole canvas:

// Form 1: Draw at (x, y) using the image's own pixel dimensions
ctx.drawImage(img, x, y);

// Form 2: Draw at (x, y) and scale to (width × height)
ctx.drawImage(img, x, y, width, height);

// Example: stretch/fill the entire canvas, no matter the photo size
ctx.drawImage(img, 0, 0, canvas.width, canvas.height);

💡 Form 2 is almost always what you want

Real photos are all different sizes — someone's phone camera might produce a 4032 × 3024 pixel image, far bigger than your 800 × 500 canvas. Using drawImage(img, 0, 0, canvas.width, canvas.height) always fits the photo to your canvas, regardless of its original resolution.

Loading from a user's file — the FileReader

For our visualiser, the client uploads their own photo from their phone or computer. The browser's FileReader API reads the file and converts it into a data URL — a long string of text that encodes the entire image. We then set that string as the src of our Image object, and the normal onload process takes over.

There are three layers of nesting here, each waiting for the previous to finish: (1) wait for the user to choose a file, (2) wait for FileReader to read it, (3) wait for the Image to decode it. This is called asynchronous code — it's one of the most important patterns in JavaScript:

// HTML: <input type="file" id="photoInput" accept="image/*">

let input = document.getElementById('photoInput');

// Layer 1: user picks a file
input.addEventListener('change', function(e) {
    let file   = e.target.files[0];    // the chosen file object
    let reader = new FileReader();

    // Layer 2: FileReader finishes reading the file
    reader.onload = function(event) {
        let img = new Image();

        // Layer 3: Image finishes decoding
        img.onload = function() {
            ctx.drawImage(img, 0, 0, canvas.width, canvas.height);
        };
        img.src = event.target.result;  // the data URL string
    };

    reader.readAsDataURL(file);  // start reading (triggers Layer 2)
});

🎨 Layers & Opacity

Draw on top of images and control how transparent the overlay is.

The Canvas is a Stack of Paint

Everything you draw on a canvas stacks on top of what was drawn before — like layers of paint or like layers in Photoshop. In our visualiser the photo goes down first, then the tattoo design goes on top.

The key to making it look realistic is opacity: making the tattoo design semi-transparent so the skin texture shows through, just like real ink that's slightly translucent. And because skin comes in many different tones, the correct opacity will look different on very fair skin versus very dark skin.

Controlling Transparency — Step by Step

ctx.globalAlpha — sets transparency for everything drawn after it

Set globalAlpha to a number between 0 (completely invisible) and 1 (fully opaque — the default). Every drawing call after this line is affected until you change it back.

Think of it like putting on sunglasses before you draw: everything you draw while wearing them is tinted. You have to take the glasses off (reset to 1) when you're done, or everything else will be tinted too.

// Default — fully solid (no transparency)
ctx.globalAlpha = 1;

// 50% transparent — like seeing through frosted glass
ctx.globalAlpha = 0.5;

// 20% opacity — very faint, just a ghost of the design
ctx.globalAlpha = 0.2;

// After drawing, ALWAYS reset to 1 — or everything stays faded!
ctx.globalAlpha = 1;

⚠️ The Most Common Bug: Forgetting to Reset

If you don't reset globalAlpha = 1, every future drawing operation will also be transparent — including the background photo on your next redraw() call. Your app will gradually fade to nothing. Always reset it immediately after your transparent drawing.

The layer order — always draw background first, overlay second

Canvas has no layers system like Photoshop — it's just paint on top of paint. Later drawing calls cover earlier ones. Our redraw() function must always follow this exact order: clear everything, paint the photo, then paint the tattoo design on top. If you do it the wrong way around, the photo will cover the tattoo.

function redraw() {
    // Step 1: Wipe the canvas clean (removes previous frame)
    ctx.clearRect(0, 0, canvas.width, canvas.height);

    // Step 2: Layer 1 — background photo at full opacity
    ctx.globalAlpha = 1;
    ctx.drawImage(backgroundPhoto, 0, 0, canvas.width, canvas.height);

    // Step 3: Layer 2 — tattoo design on top, semi-transparent
    ctx.globalAlpha = 0.75;   // ← simulates real ink transparency
    ctx.drawImage(tattooDesign, tattooX, tattooY, tattooW, tattooH);

    // Step 4: Reset — NEVER skip this
    ctx.globalAlpha = 1;
}

💜 Why call clearRect every frame?

When the user drags the tattoo, we call redraw() many times per second. Without clearing first, each redraw paints on top of the last, leaving a smeared trail of tattoo images everywhere. clearRect wipes the slate clean each time.

ctx.filter — make the tattoo look like real ink

CSS filters work on canvas drawings too. grayscale(1) converts coloured emoji into pure black/grey tones, which is much more realistic for black-ink tattoos. contrast(1.2) sharpens the edges slightly, matching how ink sits in skin:

// Apply filter BEFORE drawing the tattoo
ctx.filter = 'grayscale(1) contrast(1.2)';
ctx.drawImage(tattooDesign, x, y, w, h);

// ALWAYS reset after — otherwise all future drawing is filtered
ctx.filter = 'none';

globalCompositeOperation — blend modes for realistic ink

This is the secret weapon for photorealism. By default, drawing on canvas just paints over what's below (called 'source-over'). The 'multiply' blend mode works differently: it mathematically multiplies the colours together, so dark tattoo ink darkens the skin below it — exactly how real ink behaves in skin tissue.

On a dark skin tone, multiply makes the design naturally darker and more saturated. On a light skin tone it makes it more visible. It automatically adapts — which is why professional photo editing tools use it.

// 'multiply': dark pixels darken the skin, white pixels disappear
// This looks FAR more realistic than the default 'source-over'
ctx.globalCompositeOperation = 'multiply';
ctx.globalAlpha = 0.85;
ctx.drawImage(tattooDesign, x, y, w, h);

// Reset BOTH properties — always!
ctx.globalCompositeOperation = 'source-over';  // ← the default
ctx.globalAlpha = 1;

🖱️ Mouse & Touch Interaction

Let users drag the tattoo to exactly where they want it.

Making Things Draggable

Dragging has three stages: the user presses down on the element (mousedown), moves the mouse while holding (mousemove), then releases (mouseup). We track these three events and update the element's position during the move phase.

We also need to know whether to interpret a mouse move as a drag — we only drag if the button is held down. A boolean variable called isDragging acts as a switch that we flip on at mousedown and off at mouseup.

Building a Drag System — Step by Step

Get the exact mouse position relative to the canvas

Here's a subtle problem: mouse coordinates from events (e.clientX, e.clientY) are measured from the top-left corner of the browser window — not from the top-left of your canvas. If the canvas is 200px from the left of the window, and the user clicks 300px from the left, the actual canvas position is 100px, not 300px.

There's another problem: if the canvas is scaled by CSS (e.g. max-width: 100% shrinks it on mobile), a click at CSS pixel 150 might correspond to canvas pixel 300. We need to account for this scale factor too. This helper function handles both problems correctly:

function getPos(e) {
    // rect gives us the canvas's position and CSS size in the window
    let rect   = canvas.getBoundingClientRect();

    // Scale factor: ratio of real canvas pixels to CSS pixels
    // e.g. if canvas is 800px wide but CSS shows it at 400px, scaleX = 2
    let scaleX = canvas.width  / rect.width;
    let scaleY = canvas.height / rect.height;

    // Works for both mouse events and touch events (same function!)
    let clientX = e.touches ? e.touches[0].clientX : e.clientX;
    let clientY = e.touches ? e.touches[0].clientY : e.clientY;

    return {
        x: (clientX - rect.left) * scaleX,
        y: (clientY - rect.top)  * scaleY
    };
}

💜 Why does this function handle touch events too?

On a phone or tablet, there are no mouse events — only touch events. Touch events have a touches array with coordinates. By checking e.touches first, this one function works for both mouse and touch with no code duplication.

Set up the state variables

Before we add any event listeners, we declare variables to track the drag state. These live outside the event handler functions so all handlers can share them:

let tattooX     = 100;    // current top-left X of the tattoo
let tattooY     = 100;    // current top-left Y of the tattoo
let isDragging  = false;  // are we currently mid-drag?
let dragOffsetX = 0;     // where inside the tattoo did the click land?
let dragOffsetY = 0;

💡 Why track dragOffset?

Without an offset, the tattoo's top-left corner would snap to the cursor the moment you click — the design would jump. The offset records where inside the tattoo you clicked, and subtracts that during drag, so the tattoo moves smoothly without jumping.

Example: you click 30px from the tattoo's left edge. dragOffsetX = 30. As you drag to position 200, tattooX = 200 - 30 = 170. The tattoo's left edge stays 30px to the left of your cursor — exactly where it was when you clicked.

Handle mousedown — start dragging

When the mouse button goes down, we check whether the click landed inside the tattoo's bounding rectangle. If yes, we flip isDragging to true and record the offset. If the click missed the tattoo, nothing happens:

canvas.addEventListener('mousedown', function(e) {
    let pos = getPos(e);

    // Hit test: is the click inside the tattoo rectangle?
    // Tattoo occupies (tattooX, tattooY) to (tattooX+size, tattooY+size)
    if (pos.x > tattooX           &&
        pos.x < tattooX + tattooSize &&
        pos.y > tattooY           &&
        pos.y < tattooY + tattooSize) {

        isDragging  = true;
        dragOffsetX = pos.x - tattooX;  // ← how far inside we clicked
        dragOffsetY = pos.y - tattooY;
        canvas.style.cursor = 'grabbing'; // visual feedback
    }
});

Handle mousemove — move the tattoo

This event fires constantly as the mouse moves across the canvas. The first line guards against movement when no drag is active — we only update position if isDragging is true:

canvas.addEventListener('mousemove', function(e) {
    if (!isDragging) return;  // early exit if not dragging

    let pos = getPos(e);

    // New tattoo position: current mouse minus where we grabbed it
    tattooX = pos.x - dragOffsetX;
    tattooY = pos.y - dragOffsetY;

    redraw();  // repaint the entire canvas with the new position
});

Handle mouseup and mouseleave — stop dragging

mouseup fires when the button is released. mouseleave fires when the cursor leaves the canvas area — without it, if you drag fast and the cursor exits the canvas while held, dragging would never stop:

function stopDrag() {
    isDragging          = false;
    canvas.style.cursor = 'grab';  // back to the "ready to grab" cursor
}

canvas.addEventListener('mouseup',    stopDrag);
canvas.addEventListener('mouseleave', stopDrag); // safety net

// Touch events work the same way:
canvas.addEventListener('touchstart', startDrag, { passive: false });
canvas.addEventListener('touchmove',  doDrag,    { passive: false });
canvas.addEventListener('touchend',   stopDrag);

💡 Why { passive: false } for touch events?

By default, browsers try to scroll the page when you touch-drag. passive: false tells the browser we might call e.preventDefault() inside the handler to stop that scrolling. Without it, the page would scroll instead of dragging the tattoo on mobile.

  • Why do we need getBoundingClientRect()? — To convert window-level coordinates into canvas-level coordinates, accounting for the canvas's position on the page
  • What is the scale factor (canvas.width / rect.width) for? — Corrects for CSS scaling: a 800px canvas shown at 400px wide has scale 2, so CSS coordinates must be doubled
  • What is the drag offset for? — Prevents the tattoo jumping to the cursor; keeps it in its original position relative to where you clicked

Set up state variables and canvas context

These variables hold the current state of the whole application. Any function can read or modify them. skinBase and skinShade store the currently selected skin tone colours for drawing the placeholder background:

inside <script> 
// Canvas setup — always needed first
let canvas = document.getElementById('myCanvas');
let ctx    = canvas.getContext('2d');

// Background photo (null until the user uploads one)
let bgPhoto = null;

Write the redraw() function

This is the heart of the application. Every time anything changes — a slider moves, the user drags, a new design is selected — we call redraw() to repaint the canvas from scratch. Notice how it first reads the current slider values before doing anything:

inside <script> 
function redraw() {
    // Read current slider values (these may have changed since last call)
    tattooSize = parseInt(document.getElementById('size').value);
    let opacity = parseInt(document.getElementById('opacity').value) / 100;

    // Step 1: Clear previous frame entirely
    ctx.clearRect(0, 0, canvas.width, canvas.height);

    // Step 2: Draw background (photo OR placeholder skin tone)
    ctx.globalAlpha = 1;

    if (bgPhoto) {
        // User has uploaded a photo — show it
        ctx.drawImage(bgPhoto, 0, 0, canvas.width, canvas.height);

    } else {
        // No photo yet — draw a skin-textured placeholder
        // Use a gradient to suggest 3D skin surface
        let grd = ctx.createLinearGradient(0, 0, canvas.width, canvas.height);
        grd.addColorStop(0,   skinBase);
        grd.addColorStop(0.5, skinShade);
        grd.addColorStop(1,   skinBase);
        ctx.fillStyle = grd;
        ctx.fillRect(0, 0, canvas.width, canvas.height);

        // Instruction text on the placeholder
        ctx.fillStyle = 'rgba(0,0,0,0.3)';
        ctx.font = '16px Arial';
        ctx.textAlign = 'center';
        ctx.fillText(
            'Upload a photo to see the real effect',
            canvas.width / 2, canvas.height / 2
        );
        ctx.textAlign = 'left';
    }

    // Step 3: Draw the tattoo design on top
    ctx.globalAlpha = opacity;
    ctx.font        = tattooSize + 'px serif';
    ctx.textAlign   = 'center';
    ctx.fillText(currentDesign, tattooX + tattooSize/2, tattooY + tattooSize * 0.85);
    ctx.textAlign   = 'left';

    // Step 4: ALWAYS reset opacity — never skip this!
    ctx.globalAlpha = 1;
}

Add the drag system

The coordinate helper function converts browser window coordinates to canvas coordinates, handling both CSS scaling and the canvas position on the page:

inside <script> 
// Convert window coordinates → canvas coordinates (handles CSS scaling)
function getPos(e) {
    let rect  = canvas.getBoundingClientRect();
    let sx    = canvas.width  / rect.width;
    let sy    = canvas.height / rect.height;
    let cx    = e.touches ? e.touches[0].clientX : e.clientX;
    let cy    = e.touches ? e.touches[0].clientY : e.clientY;
    return { x: (cx - rect.left) * sx, y: (cy - rect.top) * sy };
}

function startDrag(e) {
    let pos = getPos(e);
    // Hit test: did the click/touch land inside the tattoo box?
    if (pos.x > tattooX && pos.x < tattooX + tattooSize &&
        pos.y > tattooY && pos.y < tattooY + tattooSize) {
        isDragging  = true;
        dragOffsetX = pos.x - tattooX;
        dragOffsetY = pos.y - tattooY;
        canvas.style.cursor = 'grabbing';
    }
}

function doDrag(e) {
    if (!isDragging) return;
    e.preventDefault();  // stop page scrolling on touch
    let pos = getPos(e);
    tattooX  = pos.x - dragOffsetX;
    tattooY  = pos.y - dragOffsetY;
    redraw();
}

function stopDrag() {
    isDragging          = false;
    canvas.style.cursor = 'grab';
}

// Mouse events
canvas.addEventListener('mousedown',  startDrag);
canvas.addEventListener('mousemove',  doDrag);
canvas.addEventListener('mouseup',    stopDrag);
canvas.addEventListener('mouseleave', stopDrag);

// Touch events (for mobile/tablet)
canvas.addEventListener('touchstart', startDrag, { passive: false });
canvas.addEventListener('touchmove',  doDrag,     { passive: false });
canvas.addEventListener('touchend',   stopDrag);

🚀 Challenges — Take It Further

Challenge 1 — Realistic ink blend mode

Add a toggle button. When ON, add ctx.globalCompositeOperation = 'multiply' before drawing the tattoo, and reset to 'source-over' after. Compare the result on a real uploaded photo on different skin tones — you'll immediately see why professional tools use this mode.

Challenge 2 — Rotation slider

Add a rotation range input. Before drawing the tattoo, use ctx.save(), translate to the tattoo centre, ctx.rotate(angle), draw at offset coordinates, then ctx.restore():

Rotation hint 
let angle = rotationSlider.value * Math.PI / 180;
let cx    = tattooX + tattooSize / 2;  // centre of tattoo
let cy    = tattooY + tattooSize / 2;

ctx.save();
ctx.translate(cx, cy);            // move origin to tattoo centre
ctx.rotate(angle);                  // rotate around that origin
ctx.fillText(currentDesign, -tattooSize/2, tattooSize*0.35);
ctx.restore();                      // undo transform — never forget!

Challenge 3 — Upload a custom PNG stencil

Add a second file input that accepts PNG/SVG files. Load it as an Image object and draw it with ctx.drawImage() instead of emoji text. This is how real tattoo artists would load their own hand-drawn stencil designs scanned from paper.

Challenge 4 — Skin tone aware opacity

Darker skin tones need higher ink opacity to be visible. When the user selects a darker skin tone, automatically adjust the opacity slider to a higher value (e.g., Type VI → set opacity to 90%). This simulates how tattoo artists adjust their technique for different skin tones.

💡 Something Not Working?

  • Photo not appearing: Check the browser console (F12). A CORS error appears if loading from a URL — always upload from your device instead
  • Dragging not working: The click must land inside the tattoo's bounding box. Try increasing the size slider and clicking directly on the emoji
  • Design buttons not highlighting: Make sure you're passing this in the HTML onclick: onclick="setDesign('🌹', this)"
  • Save file is blank: Call redraw() just before saving, and make sure bgPhoto was drawn with globalAlpha = 1
  • Canvas gradually fades: You forgot to reset ctx.globalAlpha = 1 — search every place you set globalAlpha and add the reset immediately after

⚠️ The Five Most Common Bugs

  • Setting img.src before img.onload — the image loads before your handler attaches; always set onload first
  • Forgetting ctx.globalAlpha = 1 — every future drawing operation becomes transparent; your canvas gradually fades to nothing
  • Forgetting ctx.filter = 'none' — all future drawing gets the same filter applied permanently
  • Forgetting ctx.restore() after ctx.save() — transforms stack and corrupt over multiple redraws
  • Using event.target instead of passing this — event.target is unreliable across browsers; always pass this from the onclick attribute
